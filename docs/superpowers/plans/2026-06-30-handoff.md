# Handoff Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A `user-invocable` `handoff` skill + `/handoff` command that auto-drafts a project-shaped `.handoffs/HANDOFF.md` at end of session, with dual discovery (a `SessionStart` hook for Claude and an `AGENTS.md` pointer for other harnesses).

**Architecture:** Five artifacts in the dotfiles repo: a `SessionStart` hook script, its registration in the templated global `settings.json`, a `bootstrap.sh` loop that symlinks commands (currently missing), the skill, and the thin command. The skill is pure instructions (executed by Claude at runtime); the hook is the only piece with executable logic, so it gets real shell tests. Everything else is verified structurally (valid JSON, syntax, resolvable symlinks, markdownlint-clean).

**Tech Stack:** Bash (hook + bootstrap), JSON (settings), Markdown with YAML frontmatter (skill + command). Formatting is handled by the repo's dprint PostToolUse hook on every Write/Edit.

## Global Constraints

- Skills live in `claude/skills/<name>/SKILL.md`; symlinked into `~/.claude/skills/<name>` by the existing per-skill loop in `bootstrap.sh`.
- Commands live in `claude/commands/<name>.md`; symlinked into `~/.claude/commands/<name>.md`.
- Hooks live in `claude/hooks/`; the whole dir is symlinked to `~/.claude/hooks` (`link_dir`), so a new script appears live immediately ŌĆö no relink needed.
- `~/.claude/settings.json` is **generated** at bootstrap via `sed "s|__HOME__|$HOME|g" claude/settings.json`, not symlinked. Use the literal token `__HOME__` in the tracked source; regenerate to apply.
- Markdown rules (markdownlint, enforced in-editor): every fenced code block has a language; no bare URLs (`<...>` or `[label](...)`); space after heading hashes. dprint auto-formats blank lines / trailing whitespace / final newline on save.
- The skill must **never run git** (no commit/branch/push) ŌĆö per the user's standing "ask before any git operation" rule. It writes files and reports they are ready to commit.
- Per-repo storage: `.handoffs/HANDOFF.md` (current) + `.handoffs/archive/YYYY-MM-DD-HHMM.md` (history). Committed to git, not ignored.
- The `## Ō¢Č Resume here` heading is a fixed contract between the skill (writes it) and the hook (extracts it). Keep it byte-for-byte identical in both.
- Work happens on the `handoff-skill` branch.

### Task 1: SessionStart hook script

**Files:**

- Create: `claude/hooks/handoff-detect.sh`

**Interfaces:**

- Consumes: SessionStart payload on stdin (JSON; we only read `.cwd`, optional). Reads `<project>/.handoffs/HANDOFF.md` and its `## Ō¢Č Resume here` section.
- Produces: on stdout, a short notice (added to session context by the SessionStart mechanism). Nothing when no handoff exists.

- [ ] **Step 1: Write the hook script**

Create `claude/hooks/handoff-detect.sh` with exactly:

```bash
#!/usr/bin/env bash
# SessionStart hook: if the current repo has a waiting handoff, surface it to
# the new session. Silent (exit 0, no output) when there is none, so it costs
# nothing in the common case. SessionStart stdout is added to session context.
set -euo pipefail

# We only need the project dir. Hooks run with cwd = project dir, so $PWD is the
# reliable default; refine from the stdin JSON's cwd only if jq is available.
input="$(cat 2>/dev/null || true)"
proj="$PWD"
if command -v jq >/dev/null 2>&1 && [ -n "$input" ]; then
  cwd="$(printf '%s' "$input" | jq -r '.cwd // empty' 2>/dev/null || true)"
  [ -n "$cwd" ] && proj="$cwd"
fi

handoff="$proj/.handoffs/HANDOFF.md"
[ -f "$handoff" ] || exit 0

# Extract the "Ō¢Č Resume here" section: from that heading to the next "## ".
resume="$(awk '
  /^##[[:space:]]+Ō¢Č[[:space:]]+Resume here[[:space:]]*$/ { f=1; next }
  /^##[[:space:]]/ { if (f) exit }
  f { print }
' "$handoff")"

echo "­¤ōŗ A session handoff is waiting in this repo."
echo "Read .handoffs/HANDOFF.md before starting ŌĆö it is the previous session's state."
if [ -n "$(printf '%s' "$resume" | tr -d '[:space:]')" ]; then
  echo
  echo "Ō¢Č Resume here:"
  printf '%s\n' "$resume"
fi
exit 0
```

# handoff ŌĆö capture session state for the next session

At the end of a working session, write a structured handoff so the next session
picks up cleanly. **Draft automatically** from what you already know, then let
the user tweak. Write files only ŌĆö **never run git** (no commit, branch, or
push); finish by telling the user the files are ready to commit.

## What you produce

In the current repo:

- `.handoffs/HANDOFF.md` ŌĆö the current handoff; the next session reads this first.
- `.handoffs/archive/YYYY-MM-DD-HHMM.md` ŌĆö the previous HANDOFF.md, archived each run.

Plus an idempotent pointer block in the repo's `AGENTS.md` so non-Claude
harnesses discover the handoff.

## Workflow

1. **Pick the profile.** Match the current repo against the table below (repo
   name first, then content heuristics). Fall back to `generic`.
2. **Gather state** without interrogating the user ŌĆö ask only if something
   critical is genuinely unknown:
   - what this session did, decided, and tried;
   - for code repos, a git snapshot: `git branch --show-current`,
     `git status --short`, `git log --oneline -5`;
   - the existing `.handoffs/HANDOFF.md` if present ŌĆö carry forward its
     still-open next-steps and questions (continuity).
3. **Archive.** If `.handoffs/HANDOFF.md` exists, move it to
   `.handoffs/archive/<YYYY-MM-DD-HHMM>.md` (create the dir if needed).
4. **Write** the new `.handoffs/HANDOFF.md` using the structure below.
5. **Maintain the AGENTS.md pointer** (see below).
6. **Show & report.** Show the draft; let the user edit or accept. Report the
   path and that `.handoffs/` is ready to commit. Do not commit.
